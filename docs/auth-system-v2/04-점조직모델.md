# 점조직 분산 운영 모델

## 개요

Google Apps Script의 동시접속 제한(약 100명)을 해결하기 위해, 각 교사가 자신만의 독립된 시스템을 소유하고 운영하는 분산 구조를 채택한다.

---

## 핵심 철학

> "각 교사가 자신만의 독립된 시스템을 소유하고 운영한다"

## 결정 사항

### 1. 설치 방식: 원클릭 + 수동 가이드 병행

**주 경로: 원클릭 설치**
```
1. 공식 사이트에서 "내 시스템 만들기" 클릭
2. 구글 로그인 + 권한 승인
3. 자동으로 모든 설정 완료
4. 내 URL 받기
```

**보조 경로: 수동 설치 가이드**
- 원클릭이 안 되는 환경용
- 구조 이해하고 싶은 교사용

### 2. 업데이트 정책: 알림 + 중요 업데이트 강조

**일반 업데이트**
- 알림만 표시
- 교사가 선택적으로 적용
- 새 기능, 사소한 개선

**중요 업데이트**
- 강력 권고 + 지속적 알림
- 보안 패치, 심각한 버그 수정
- 업데이트 전까지 경고 유지

```
[일반 업데이트 알림]
┌─────────────────────────────────────────┐
│  💡 새 버전 v2.1 (선택)                 │
│  [ 자세히 보기 ]  [ 업데이트 ]  [ ✕ ]  │
└─────────────────────────────────────────┘

[중요 업데이트 알림]
┌─────────────────────────────────────────┐
│  ⚠️ 중요 보안 업데이트 v2.2            │
│                                         │
│  보안 문제가 발견되어 업데이트가        │
│  필요합니다.                            │
│                                         │
│  [ 지금 업데이트 ]                      │
│                                         │
│  ⚠️ 업데이트 전까지 이 알림이 유지됩니다│
└─────────────────────────────────────────┘
```

### 3. 학생 데이터 이관: 교사 재량 + 도구 제공

**기본 원칙**
- 시스템 간 독립 (자동 연동 없음)
- 필요시 도구 활용

**제공 도구**

| 대상 | 기능 |
|------|------|
| 교사용 | 학생 데이터 내보내기 (CSV) |
| 교사용 | 학생 데이터 가져오기 (CSV) |
| 교사용 | 작품 일괄 내보내기 (ZIP) |
| 학생용 | 내 작품 다운로드 (PDF/이미지) |
| 학생용 | 내 작품 구글 드라이브 저장 |

### 4. 교사 간 협업: 완전 독립 + 구글 공유 안내

**기본 원칙**
- 각 시스템 완전 독립
- 연동 기능 없음

**협업 필요시**
- 구글 기본 기능 활용 (스프레드시트 공유)
- 협업 가이드 제공

### 5. 중앙 허브: 랜딩 페이지 + 선택적 커뮤니티

**1단계 (초기)**
- 간단한 랜딩 페이지
- 설치 버튼
- 가이드, FAQ
- GitHub Issues로 문의

**2단계 (사용자 증가 시)**
- FAQ 확충
- 오픈채팅방 개설

**3단계 (커뮤니티 필요시)**
- 네이버 카페 또는 밴드 개설

```
┌─────────────────────────────────────────┐
│  📚 스토리 구성 웹학습지                │
│                                         │
│  선생님만의 학습지를 만들어보세요!      │
│                                         │
│  [ 🚀 내 시스템 만들기 ]                │
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  ✅ 무료  ✅ 내 드라이브 저장  ✅ 광고없음│
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  📖 사용 가이드                         │
│  ❓ 자주 묻는 질문                      │
│  💬 문의하기                            │
│                                         │
└─────────────────────────────────────────┘
```

### 6. 기술 지원: 단계별 지원 체계

```
[1단계] 셀프 서비스
  - FAQ
  - 가이드 문서
  - 문제 해결 가이드
      ↓ 해결 안 되면

[2단계] 커뮤니티 지원 (추후)
  - 오픈채팅 또는 카페
  - 사용자 상호 도움
      ↓ 해결 안 되면

[3단계] 직접 문의
  - 구글 폼 또는 이메일
  - 버그 리포트
```

## 장점

| 항목 | 설명 |
|------|------|
| 동시접속 | 문제 완전 해결 (각 시스템 독립) |
| 데이터 소유권 | 명확함 (교사 본인) |
| 서버 비용 | 없음 (구글 무료 티어) |
| 장애 격리 | 한 시스템 문제가 다른 시스템에 영향 없음 |
| 개인정보보호 | 데이터 컨트롤러 명확 (교사) |
| 학생 가입 | 대폭 단순화 |

## 점조직 모델에서 단순화되는 것들

### 학생 가입

| 기존 (중앙 집중형) | 점조직 |
|-------------------|--------|
| 전역 고유 학생 ID 필요 | 시스템 내 이름+번호로 충분 |
| 교사 코드 입력 | URL 자체가 식별자 |
| 기존 학생 자동 매칭 | 불필요 (같은 시스템 내) |
| 복잡한 동의 체계 | 교사 재량 |

### 학생 가입 흐름

```
1️⃣ 선생님이 알려준 URL 접속 (또는 QR)
      ↓
2️⃣ 이름 + 번호 입력
      ↓
3️⃣ PIN 만들기 (4자리)
      ↓
4️⃣ 완료!
```

## 교사 시스템 구성

```
김영희 선생님의 시스템
├── 📊 스프레드시트 (데이터 저장)
│   ├── 학생 시트
│   ├── 작품 시트
│   └── 설정 시트
│
├── 📜 Apps Script (백엔드)
│   └── Code.gs (사본)
│
├── 🌐 웹앱 URL (프론트엔드)
│   └── https://script.google.com/.../exec
│
└── 👨‍🎓 학생들
    └── 김영희 선생님 URL로 접속
```

## 운영 체계

```
[공식 사이트] ─────────────────────────────
    │
    ├── 🚀 원클릭 설치
    ├── 📖 사용 가이드 / FAQ
    └── 💬 문의 (단계별)

[각 교사 시스템] ──────────────────────────
    │
    ├── 완전 독립 운영
    ├── 업데이트 알림 수신 (선택적 적용)
    └── 필요시 구글 공유로 협업
```
