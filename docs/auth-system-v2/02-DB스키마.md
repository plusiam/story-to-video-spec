# 데이터베이스 스키마 설계 (점조직 모델)

## 개요

점조직 모델에서는 각 교사가 독립된 스프레드시트를 소유한다. 중앙 집중형 대비 스키마가 대폭 단순화된다.

---

## 점조직 모델의 특징

### 기존 (중앙 집중형) vs 점조직

| 구분 | 중앙 집중형 | 점조직 |
|------|------------|--------|
| 스프레드시트 | 1개 (공용) | 교사별 각 1개 |
| 학생 ID | 전역 고유 ID 필요 | 시스템 내 이름+번호 |
| 학급 관리 | 복잡한 ENROLLMENTS 테이블 | 불필요 (단일 교사) |
| 교사 관리 | TEACHERS 테이블 필요 | 불필요 (소유자 = 교사) |

---

## 단순화된 테이블 구조

### 필요한 시트 (4개)

```
교사의 스프레드시트
├── 📋 STUDENTS (학생)
├── 📋 WORKS_STEP1 (1단계 작품)
├── 📋 WORKS_STEP2 (2단계 작품)
├── 📋 WORKS_STEP3 (3단계 작품)
└── 📋 SETTINGS (설정) - 선택
```

---

## 테이블 상세 설계

### 1. STUDENTS (학생)

교사 시스템 내 학생 정보를 저장한다.

| 컬럼명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| name | STRING | ✓ | 이름 |
| number | NUMBER | ✓ | 번호 |
| pinHash | STRING | | PIN 해시 (SHA-256), 미설정 시 빈값 |
| token | STRING | ✓ | QR 인증용 고유 토큰 |
| createdAt | DATETIME | ✓ | 등록일 (교사가 등록한 날) |
| lastAccessAt | DATETIME | | 마지막 접속 |
| status | STRING | ✓ | pending / active / inactive |

**상태 설명:**
| 상태 | 설명 | PIN | QR 사용 |
|------|------|-----|---------|
| pending | 교사 등록, PIN 미설정 | 없음 | ❌ |
| active | PIN 설정 완료 | 있음 | ✅ |
| inactive | 교사가 비활성화 | 유지 | ❌ |

**헤더 정의:**
```javascript
const STUDENT_HEADERS = [
  "이름",        // [0]
  "번호",        // [1]
  "PIN해시",     // [2] 빈값 가능 (pending 상태)
  "토큰",        // [3] QR 인증용
  "등록일",      // [4]
  "마지막접속",  // [5]
  "상태"         // [6] pending / active / inactive
];
```

**예시 데이터:**
```
| 이름   | 번호 | PIN해시    | 토큰       | 등록일     | 마지막접속 | 상태    |
|--------|------|------------|------------|------------|------------|---------|
| 홍길동 | 15   | a1b2c3...  | tk_abc123  | 2026-03-02 | 2026-03-10 | active  |
| 김서연 | 2    |            | tk_def456  | 2026-03-02 |            | pending |
| 박지훈 | 3    | d4e5f6...  | tk_ghi789  | 2026-03-02 | 2026-03-09 | active  |
```

### 2. WORKS_STEP1 (1단계 작품)

4컷 스토리 작품을 저장한다.

| 컬럼명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| studentName | STRING | ✓ | 학생 이름 |
| studentNumber | NUMBER | ✓ | 학생 번호 |
| payload | JSON | ✓ | 작품 데이터 |
| createdAt | DATETIME | ✓ | 생성일 |
| updatedAt | DATETIME | ✓ | 수정일 |
| isComplete | BOOLEAN | | 완료 여부 |
| status | STRING | ✓ | draft / submitted / published |

**헤더 정의:**
```javascript
const WORK_STEP1_HEADERS = [
  "학생이름",    // [0]
  "학생번호",    // [1]
  "작품데이터",  // [2] JSON
  "생성일",      // [3]
  "수정일",      // [4]
  "완료여부",    // [5]
  "상태"         // [6]
];
```

### 5. SETTINGS (설정) - 선택

교사 시스템 설정을 저장한다.

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| key | STRING | 설정 키 |
| value | STRING | 설정 값 |

**예시:**
```

```

## 삭제된 테이블 (중앙 집중형 대비)

점조직 모델에서 **불필요해진 테이블들**:

| 테이블 | 이유 |
|--------|------|
| TEACHERS | 시스템 소유자 = 교사 (별도 관리 불필요) |
| CLASSES | 단일 교사 시스템 (학급 구분 불필요) |
| ENROLLMENTS | 학생↔학급 관계 불필요 |
| CONSENTS | 교사 재량 (시스템 외부에서 관리) |

---

## 데이터 관계

```
[단순한 구조]

STUDENTS
    │
    └──→ WORKS_STEP1
    └──→ WORKS_STEP2
    └──→ WORKS_STEP3

연결 키: 학생이름 + 학생번호
```

---

## 학생 식별 방식

### 기존 (중앙 집중형)
```
studentId: "STU_a1b2c3d4" (전역 고유)
```

### 점조직
```
학생이름 + 학생번호 조합으로 식별
예: "홍길동" + 15 → 시스템 내 유일
```

**장점:**
- UUID 생성 불필요
- 교사가 직관적으로 이해
- 스프레드시트에서 직접 확인 가능

**제약:**
- 같은 반에 동명이인 + 같은 번호는 불가 (현실적으로 없음)

## 코드 예시

### 교사: 학생 등록 (PIN 미설정)

```javascript
function registerStudentByTeacher(name, number, pin = null) {
  const sheet = SpreadsheetApp.getActive().getSheetByName('STUDENTS');

  // 중복 확인 (이름 + 번호)
  const existing = findStudent(name, number);
  if (existing) {
    return { success: false, error: '이미 등록된 학생입니다.' };
  }

  // 토큰 생성 (QR용)
  const token = 'tk_' + Utilities.getUuid().substring(0, 12);
  const now = new Date();

  // PIN 처리
  let pinHash = '';
  let status = 'pending';

  if (pin) {
    // 교사가 PIN 지정한 경우
    pinHash = hashPin(pin);
    status = 'active';
  }

  // 저장: 이름, 번호, PIN해시, 토큰, 등록일, 마지막접속, 상태
  sheet.appendRow([name, number, pinHash, token, now, '', status]);

  return { success: true, token: token, status: status };
}
```

### 학생: PIN 설정 (첫 접속)

```javascript
function setStudentPin(name, number, pin) {
  const student = findStudent(name, number);

  if (!student) {
    return { success: false, error: '등록되지 않은 학생입니다.' };
  }

  if (student.status !== 'pending') {
    return { success: false, error: '이미 PIN이 설정되어 있습니다.' };
  }

  const sheet = SpreadsheetApp.getActive().getSheetByName('STUDENTS');
  const pinHash = hashPin(pin);
  const now = new Date();

  // PIN해시, 마지막접속, 상태 업데이트
  sheet.getRange(student.row, 3).setValue(pinHash);
  sheet.getRange(student.row, 6).setValue(now);
  sheet.getRange(student.row, 7).setValue('active');

  return { success: true };
}
```

### 학생 찾기

```javascript
function findStudent(name, number) {
  const sheet = SpreadsheetApp.getActive().getSheetByName('STUDENTS');
  const data = sheet.getDataRange().getValues();

  for (let i = 1; i < data.length; i++) {
    if (data[i][0] === name && data[i][1] === number) {
      return {
        row: i + 1,
        name: data[i][0],
        number: data[i][1],
        pinHash: data[i][2],
        token: data[i][3],
        createdAt: data[i][4],
        lastAccessAt: data[i][5],
        status: data[i][6]
      };
    }
  }
  return null;
}
```

### 토큰으로 학생 찾기 (QR 로그인용)

```javascript
function findStudentByToken(token) {
  const sheet = SpreadsheetApp.getActive().getSheetByName('STUDENTS');
  const data = sheet.getDataRange().getValues();

  for (let i = 1; i < data.length; i++) {
    if (data[i][3] === token) {
      return {
        row: i + 1,
        name: data[i][0],
        number: data[i][1],
        pinHash: data[i][2],
        token: data[i][3],
        status: data[i][6]
      };
    }
  }
  return null;
}
```

### 학생 로그인 (PIN 방식)

```javascript
function loginStudent(name, number, pin) {
  const student = findStudent(name, number);

  if (!student) {
    return { success: false, error: '등록되지 않은 학생입니다.' };
  }

  if (student.status === 'pending') {
    return { success: false, error: 'PIN 설정이 필요합니다.', needSetPin: true };
  }

  if (student.status === 'inactive') {
    return { success: false, error: '비활성화된 계정입니다.' };
  }

  if (student.pinHash !== hashPin(pin)) {
    return { success: false, error: 'PIN이 올바르지 않습니다.' };
  }

  // 마지막 접속 업데이트
  updateLastAccess(student.row);

  return {
    success: true,
    name: student.name,
    number: student.number
  };
}
```

### 학생 로그인 (QR/토큰 방식)

function loginStudentByToken(token) {

    return { success: false, error: '사용할 수 없는 계정입니다.' };
  }

  // 마지막 접속 업데이트

### 교사: PIN 재설정

```javascript
function resetStudentPin(name, number, newPin) {
  const student = findStudent(name, number);

  if (!student) {
    return { success: false, error: '학생을 찾을 수 없습니다.' };
  }

  const sheet = SpreadsheetApp.getActive().getSheetByName('STUDENTS');
  const newHash = hashPin(newPin);

  sheet.getRange(student.row, 3).setValue(newHash);

  // pending이었다면 active로 변경
  if (student.status === 'pending') {
    sheet.getRange(student.row, 7).setValue('active');
  }

  return { success: true };
}
```

## 데이터 내보내기/가져오기

### 내보내기 형식 (CSV)

```csv
이름,번호,가입일,상태
홍길동,15,2026-03-02,active
김서연,2,2026-03-02,active
박지훈,3,2026-03-03,active
```

### 가져오기 함수 (교사용 일괄 등록)

```javascript
function importStudents(csvData, pinMode = 'student') {
  const rows = Utilities.parseCsv(csvData);
  const results = { success: 0, skipped: 0, errors: [] };

  // 헤더 제외하고 추가
  for (let i = 1; i < rows.length; i++) {
    const [name, number] = rows[i];

    let pin = null;
    if (pinMode === 'default') {
      // 기본 PIN: 번호 6자리
      pin = String(number).padStart(6, '0');
    }
    // pinMode === 'student'면 PIN은 null (학생이 직접 설정)

    const result = registerStudentByTeacher(name, parseInt(number), pin);
    if (result.success) {
      results.success++;
    } else {
      results.skipped++;
      results.errors.push({ name, number, error: result.error });
    }
  }

  return results;
}
```
