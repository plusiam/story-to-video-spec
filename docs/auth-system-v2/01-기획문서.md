# 학생 인증 시스템 개선 기획서 v2

## 개요

Google Apps Script 동시접속 제한(약 100명)을 해결하고, 학생 가입 절차를 단순화하기 위해 **점조직 분산 운영 모델**을 채택한다.

---

## 핵심 변경: 점조직 모델 채택

### 기존 구상
- 중앙 집중형 시스템
- 전역 학생 ID, 복잡한 매칭 로직
- 동시접속 100명 제한 문제

### 변경된 방향
- **각 교사가 독립된 시스템 운영**
- 학생 가입 대폭 단순화
- 동시접속 문제 완전 해결

```

```

## 학생 가입 정책 (교사 선등록 방식)

### 핵심 원칙

- **교사가 먼저 학생 명단을 등록해야 학생이 가입 가능**
- 충돌 방지 및 보안 강화
- PIN은 교사 지정 또는 학생 직접 설정 선택 가능

### 교사: 학생 등록

```
1️⃣ 교사가 학생 명단 등록 (이름 + 번호)
      ↓
2️⃣ PIN 설정 방식 선택
   - 학생이 첫 접속 시 직접 설정
   - 교사가 미리 지정
      ↓
3️⃣ 학생별 QR 코드 생성 (선택)
      ↓
4️⃣ URL 또는 QR 배포
```

### 학생: 가입 흐름 (PIN 미설정 상태)

```
1️⃣ 선생님이 알려준 URL 접속 (또는 반 QR 스캔)
      ↓
2️⃣ 이름 + 번호 입력
      ↓
3️⃣ 등록된 학생인지 확인
   - 있음 → PIN 설정 화면
   - 없음 → "선생님께 문의하세요"
      ↓
4️⃣ PIN 만들기 (숫자 6자리)
      ↓
5️⃣ 완료! (상태: active)
```

### 학생: 로그인 흐름 (QR 사용)

```
1️⃣ 학생별 QR 스캔
      ↓
2️⃣ 바로 입장! (PIN 불필요)
```

### 학생: 로그인 흐름 (URL 사용 - 집에서)

```
1️⃣ URL 접속
      ↓
2️⃣ 이름 + 번호 + PIN 입력
      ↓
3️⃣ 입장!
```

### 학생 상태

| 상태 | 설명 | QR 사용 |
|------|------|---------|
| pending | 교사 등록, PIN 미설정 | ❌ |
| active | PIN 설정 완료 | ✅ |
| inactive | 교사가 비활성화 | ❌ |

|-------------------|-----------|

| 학급 유효기간 관리 | ⚡ 교사가 직접 관리 |

## 교사 시스템 생성

### "가입" → "시스템 만들기"로 변경

```
1️⃣ 공식 사이트에서 "내 시스템 만들기" 클릭
      ↓
2️⃣ 구글 로그인 + 권한 승인
      ↓
3️⃣ 자동으로:
   - 스프레드시트 생성 (내 드라이브)
   - Apps Script 복사
   - 웹앱 배포
      ↓
4️⃣ 내 전용 URL 받기
      ↓
5️⃣ 학생들에게 URL 공유
```

### 교사 시스템 구성

```
김영희 선생님의 시스템
├── 📊 스프레드시트 (데이터 저장)
│   ├── 학생 시트
│   ├── 작품 시트
│   └── 설정 시트
│
├── 📜 Apps Script (백엔드)
│   └── Code.gs (사본)
│
├── 🌐 웹앱 URL (프론트엔드)
│   └── https://script.google.com/.../exec
│
└── 👨‍🎓 학생들
    └── 이 URL로 접속
```

## 점조직 모델 결정 사항

| # | 안건 | 결정 |
|---|------|------|
| 1 | 설치 방식 | 원클릭 설치 + 수동 가이드 병행 |
| 2 | 업데이트 정책 | 알림 + 중요 업데이트 강조 |
| 3 | 학생 데이터 이관 | 교사 재량 + 내보내기/가져오기 도구 |
| 4 | 교사 간 협업 | 완전 독립 + 필요시 구글 공유 |
| 5 | 중앙 허브 | 랜딩 페이지 + 선택적 커뮤니티 |
| 6 | 기술 지원 | 단계별 (FAQ → 커뮤니티 → 문의) |

## 학부모 동의

### 기본 방침
- 교사 재량으로 운영
- 동의서 템플릿 제공

### 제공 템플릿
- 기본 동의서 (이름, 학급, 작품 저장)
- 구글 연결 추가 동의서 (선택)
- 작품 공개 동의서 (선택)

### 교사 선택 사항
- 템플릿 사용 여부
- 학교 자체 양식 사용 가능
- 동의 수집 방식 (온라인/오프라인)

## 데이터 관리

### 소유권
- 데이터 = 교사 구글 드라이브에 저장
- 교사가 완전한 소유권 보유
- 삭제/수정/백업 교사 재량

### 이관 도구 (선택적)

**교사용**
- 학생 데이터 내보내기 (CSV)
- 학생 데이터 가져오기 (CSV)
- 작품 일괄 내보내기 (ZIP)

**학생용**
- 내 작품 다운로드 (PDF/이미지)
- 내 작품 구글 드라이브 저장

## 장점

| 항목 | 효과 |
|------|------|
| 동시접속 | 문제 완전 해결 |
| 학생 가입 | 3단계로 단순화 |
| 데이터 소유 | 교사에게 명확 |
| 서버 비용 | 없음 (구글 무료) |
| 개인정보 | 책임 소재 명확 |
| 장애 격리 | 시스템 간 영향 없음 |

## 사용자별 경험 요약

### 교사
```
원클릭 설치 → 학생 명단 등록 → QR/URL 배포 → 운영
```

### 학생 (첫 접속)
```
URL 접속 → 이름+번호 입력 → PIN 설정 (6자리) → 사용 시작
```

### 학생 (이후 접속 - 교실)
```
학생별 QR 스캔 → 바로 입장
```

### 학생 (이후 접속 - 집)
```
URL 접속 → 이름+번호+PIN 입력 → 입장
```

### 학부모
```
(교사 재량) 동의서 제출 요청 시 → 동의
```
