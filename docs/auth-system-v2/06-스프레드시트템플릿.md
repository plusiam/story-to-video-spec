# 스프레드시트 템플릿 설계

## 개요

교사가 "시스템 만들기"를 클릭하면 복사되는 구글 스프레드시트의 구조를 정의한다.

---

## 템플릿 구성

### 시트 목록 (5개)

```
📊 스프레드시트 템플릿
├── 📋 STUDENTS      (학생 관리)
├── 📋 WORKS_STEP1   (1단계 작품)
├── 📋 WORKS_STEP2   (2단계 작품)
├── 📋 WORKS_STEP3   (3단계 작품)
└── 📋 SETTINGS      (시스템 설정)
```

---

## 1. STUDENTS 시트

### 용도
학생 정보 저장 및 관리

### 레이아웃

```
┌────┬────────┬────┬──────────┬──────────────┬────────────┬────────────┬─────────┐
│ A  │   B    │ C  │    D     │      E       │     F      │     G      │    H    │
├────┼────────┼────┼──────────┼──────────────┼────────────┼────────────┼─────────┤
│ 1  │ 이름   │번호│ PIN해시  │    토큰      │   등록일   │ 마지막접속 │  상태   │
├────┼────────┼────┼──────────┼──────────────┼────────────┼────────────┼─────────┤
│ 2  │ (예시) │  1 │          │ tk_abc123... │ 2026-01-17 │            │ pending │
│ 3  │        │    │          │              │            │            │         │
│... │        │    │          │              │            │            │         │
└────┴────────┴────┴──────────┴──────────────┴────────────┴────────────┴─────────┘
```

### 컬럼 상세

| 컬럼 | 헤더명 | 타입 | 필수 | 설명 | 예시 |
|------|--------|------|------|------|------|
| A | 이름 | 텍스트 | ✓ | 학생 이름 | 홍길동 |
| B | 번호 | 숫자 | ✓ | 출석 번호 | 15 |
| C | PIN해시 | 텍스트 | | SHA-256 해시값, 빈값=pending | a1b2c3d4... |
| D | 토큰 | 텍스트 | ✓ | QR 인증용 고유 토큰 | tk_abc123xyz |
| E | 등록일 | 날짜 | ✓ | 교사가 등록한 날짜 | 2026-01-17 |
| F | 마지막접속 | 날짜시간 | | 최근 접속 시각 | 2026-01-17 14:30 |
| G | 상태 | 텍스트 | ✓ | pending/active/inactive | active |

### 서식 설정

```
[헤더 행 (1행)]
- 배경색: #4A90D9 (파란색)
- 글자색: 흰색
- 글꼴: 굵게
- 행 높이: 30px
- 고정 (freeze)

[데이터 영역]
- 번호 컬럼: 가운데 정렬
- 날짜 컬럼: 날짜 형식 (YYYY-MM-DD)
- 상태 컬럼: 가운데 정렬

[조건부 서식]
- 상태 = "pending" → 배경 노란색 (#FFF3CD)
- 상태 = "active" → 배경 연두색 (#D4EDDA)
- 상태 = "inactive" → 배경 회색 (#E9ECEF)
```

### 데이터 유효성 검사

```
[B열 - 번호]
- 조건: 1~100 사이 정수
- 오류 메시지: "번호는 1~100 사이 숫자여야 합니다"

[G열 - 상태]
- 조건: 목록에서 선택 (pending, active, inactive)
- 드롭다운 표시
```

## 2. WORKS_STEP1 시트

### 용도

### 레이아웃

```

│ A  │    B     │    C     │         D          │     E      │     F      │    G     │   H    │
├────┼──────────┼──────────┼────────────────────┼────────────┼────────────┼──────────┼────────┤
│ 1  │ 학생이름 │ 학생번호 │     작품데이터     │   생성일   │   수정일   │ 완료여부 │  상태  │

│... │          │          │                    │            │            │          │        │
└────┴──────────┴──────────┴────────────────────┴────────────┴────────────┴──────────┴────────┘
```

### 컬럼 상세

| 컬럼 | 헤더명 | 타입 | 필수 | 설명 |
|------|--------|------|------|------|
| A | 학생이름 | 텍스트 | ✓ | STUDENTS.이름 참조 |
| B | 학생번호 | 숫자 | ✓ | STUDENTS.번호 참조 |
| C | 작품데이터 | 텍스트 | ✓ | JSON 형식 작품 내용 |
| D | 생성일 | 날짜시간 | ✓ | 최초 저장 시각 |

| F | 완료여부 | 불리언 | | TRUE/FALSE |

## 5. SETTINGS 시트

### 용도
시스템 설정 저장 (Key-Value 형식)

### 레이아웃

```
┌────┬────────────────┬─────────────────────────────┬─────────────────────────────┐
│    │       A        │              B              │              C              │
├────┼────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 1  │      키        │             값              │            설명             │
├────┼────────────────┼─────────────────────────────┼─────────────────────────────┤
│ 2  │ teacherName    │ (설정 전)                   │ 교사 이름                   │
│ 3  │ schoolName     │ (설정 전)                   │ 학교 이름                   │
│ 4  │ className      │ (설정 전)                   │ 학급 이름 (예: 4학년 2반)   │
│ 5  │ pinSalt        │ (자동 생성됨)               │ PIN 해시용 Salt             │
│ 6  │ version        │ 1.0.0                       │ 시스템 버전                 │
│ 7  │ createdAt      │ (자동 생성됨)               │ 시스템 생성일               │
│ 8  │ lastBackup     │                             │ 마지막 백업일               │
│ 9  │ theme          │ default                     │ 테마 설정                   │
│10  │ welcomeMessage │ 열심히 공부해요!            │ 학생 환영 메시지            │
└────┴────────────────┴─────────────────────────────┴─────────────────────────────┘
```

### 설정 항목 상세

| 키 | 기본값 | 필수 | 설명 |
|-----|--------|------|------|
| teacherName | (빈값) | ✓ | 교사 이름 (첫 설정 시 입력) |
| schoolName | (빈값) | | 학교 이름 |
| className | (빈값) | | 학급명 |
| pinSalt | (자동) | ✓ | 시스템 생성 시 UUID로 자동 생성 |
| version | 1.0.0 | ✓ | 현재 설치된 버전 |
| createdAt | (자동) | ✓ | 시스템 생성 시 자동 기록 |
| lastBackup | (빈값) | | 마지막 백업 날짜 |
| theme | default | | UI 테마 (향후 확장) |
| welcomeMessage | 열심히 공부해요! | | 학생 로그인 화면 메시지 |

### 서식 설정

```
[헤더 행]
- 배경색: #9E9E9E (회색)
- 글자색: 흰색
- 글꼴: 굵게

[키 컬럼 (A열)]
- 너비: 150px
- 배경색: #F5F5F5 (연한 회색)

[값 컬럼 (B열)]
- 너비: 250px

[설명 컬럼 (C열)]
- 너비: 250px
- 글자색: #666666 (회색)
- 글꼴: 이탤릭
```

## 초기화 스크립트

시스템 생성 시 실행되는 초기화 코드:

```javascript
function initializeSpreadsheet() {
  const ss = SpreadsheetApp.getActive();

  // 1. 시트 생성 확인
  ensureSheet('STUDENTS', STUDENT_HEADERS);
  ensureSheet('WORKS_STEP1', WORK_HEADERS);
  ensureSheet('WORKS_STEP2', WORK_HEADERS);
  ensureSheet('WORKS_STEP3', WORK_HEADERS);
  ensureSheet('SETTINGS', SETTINGS_HEADERS);

  // 2. SETTINGS 초기값 설정
  initializeSettings();

  // 3. 서식 적용
  applyFormatting();

  // 4. 데이터 유효성 검사 설정
  setDataValidation();

  // 5. 시트 보호 설정
  protectSheets();
}

function ensureSheet(name, headers) {
  const ss = SpreadsheetApp.getActive();
  let sheet = ss.getSheetByName(name);

  if (!sheet) {
    sheet = ss.insertSheet(name);
  }

  // 헤더 설정
  if (sheet.getLastRow() === 0) {
    sheet.getRange(1, 1, 1, headers.length).setValues([headers]);
  }

  return sheet;
}

function initializeSettings() {
  const sheet = SpreadsheetApp.getActive().getSheetByName('SETTINGS');
  const now = new Date();
  const salt = Utilities.getUuid();

  const settings = [
    ['teacherName', '', '교사 이름'],
    ['schoolName', '', '학교 이름'],
    ['className', '', '학급 이름'],
    ['pinSalt', salt, 'PIN 해시용 Salt (자동 생성)'],
    ['version', '1.0.0', '시스템 버전'],
    ['createdAt', now, '시스템 생성일'],
    ['lastBackup', '', '마지막 백업일'],
    ['theme', 'default', '테마 설정'],
    ['welcomeMessage', '열심히 공부해요!', '학생 환영 메시지']
  ];

  // 기존 데이터 확인 후 없으면 추가
  const existing = sheet.getDataRange().getValues();
  if (existing.length <= 1) {
    sheet.getRange(1, 1, 1, 3).setValues([['키', '값', '설명']]);
    sheet.getRange(2, 1, settings.length, 3).setValues(settings);
  }
}

function applyFormatting() {
  const ss = SpreadsheetApp.getActive();

  // STUDENTS 시트 서식
  const studentsSheet = ss.getSheetByName('STUDENTS');
  formatHeader(studentsSheet, '#4A90D9');
  studentsSheet.setFrozenRows(1);

  // WORKS 시트들 서식
  ['WORKS_STEP1', 'WORKS_STEP2', 'WORKS_STEP3'].forEach((name, i) => {
    const colors = ['#7CB342', '#FFB300', '#FF7043'];
    const sheet = ss.getSheetByName(name);
    formatHeader(sheet, colors[i]);
    sheet.setFrozenRows(1);
  });

  // SETTINGS 시트 서식
  const settingsSheet = ss.getSheetByName('SETTINGS');
  formatHeader(settingsSheet, '#9E9E9E');
}

function formatHeader(sheet, color) {
  const headerRange = sheet.getRange(1, 1, 1, sheet.getLastColumn());
  headerRange
    .setBackground(color)
    .setFontColor('white')
    .setFontWeight('bold')
    .setHorizontalAlignment('center');
}
```

## 템플릿 배포 방식

### 방법 1: 공개 스프레드시트 복사

```
1. 마스터 템플릿을 "링크가 있는 모든 사용자 - 뷰어"로 공개
2. 교사가 "사본 만들기"로 복사
3. 복사된 시트에서 Apps Script 배포
```

### 방법 2: Apps Script로 자동 생성

```javascript
function createNewSystem() {
  // 1. 새 스프레드시트 생성
  const ss = SpreadsheetApp.create('📚 스토리 학습지');

  // 2. 초기화
  initializeSpreadsheet();

  // 3. 스프레드시트 ID 반환
  return ss.getId();
}
```

## 교사용 시트 직접 수정 가이드

교사가 웹 UI 대신 스프레드시트에서 직접 데이터를 관리할 때 참고하는 가이드.

### STUDENTS 시트 직접 수정

#### ✅ 수정 가능한 컬럼

| 컬럼 | 설명 | 입력 방법 |
|------|------|----------|
| 이름 | 학생 이름 | 직접 입력 (예: 홍길동) |
| 번호 | 출석 번호 | 숫자만 입력 (예: 15) |
| 상태 | 학생 상태 | pending / active / inactive 중 선택 |

#### ⚠️ 수정하면 안 되는 컬럼

| 컬럼 | 이유 | 문제 발생 시 |
|------|------|-------------|
| PIN해시 | 시스템이 암호화하여 저장 | 학생 로그인 불가 |
| 토큰 | QR 인증에 사용 | QR 코드 무효화 |
| 등록일 | 자동 기록 | 기록 손상 |
| 마지막접속 | 자동 기록 | 기록 손상 |

#### 💡 학생 일괄 등록 방법

```
[준비]
1. 엑셀에서 학생 명단 준비
   - A열: 이름
   - B열: 번호

[등록]
2. STUDENTS 시트 열기
3. 이름, 번호 컬럼에 복사-붙여넣기
4. 상태 컬럼에 'pending' 입력
5. 나머지 컬럼(PIN해시, 토큰 등)은 비워두기

[완료]
6. 웹 대시보드에서 확인
7. 시스템이 토큰 자동 생성
8. 학생에게 URL 안내
```

#### 예시: 학생 5명 일괄 등록

```
복사할 데이터 (엑셀에서):
홍길동    1
김서연    2
박지훈    3
이수민    4
정다은    5

붙여넣기 후:
| 이름   | 번호 | PIN해시 | 토큰 | 등록일 | 마지막접속 | 상태    |
|--------|------|---------|------|--------|------------|---------|
| 홍길동 | 1    |         |      |        |            | pending |
| 김서연 | 2    |         |      |        |            | pending |
| 박지훈 | 3    |         |      |        |            | pending |
| 이수민 | 4    |         |      |        |            | pending |

```

### SETTINGS 시트 직접 수정

#### ✅ 수정 가능한 설정

| 키 | 설명 | 수정 방법 |
|-----|------|----------|
| teacherName | 교사 이름 | 값 직접 입력 |
| schoolName | 학교 이름 | 값 직접 입력 |
| className | 학급명 | 값 직접 입력 |
| welcomeMessage | 환영 메시지 | 값 직접 입력 |
| theme | 테마 | default (현재 유일) |

#### ⚠️ 수정하면 안 되는 설정

| 키 | 이유 |
|-----|------|
| pinSalt | PIN 암호화에 사용, 변경 시 모든 학생 로그인 불가 |
| version | 업데이트 관리에 사용 |
| createdAt | 시스템 생성 기록 |

### WORKS_* 시트 직접 수정

#### ⛔ 직접 수정 권장하지 않음

작품 데이터는 JSON 형식으로 저장되어 있어 직접 수정 시 데이터 손상 위험이 높습니다.

**필요한 경우:**
- 웹 대시보드에서 작품 관리 기능 사용
- 학생이 직접 수정하도록 안내

**불가피한 경우 주의사항:**
```json
// 올바른 JSON 형식 유지 필수
{
  "title": "작품 제목",
  "panels": [...]
}

// 흔한 오류:
// - 따옴표 누락
// - 쉼표 누락/초과
// - 한글 깨짐
```

### 문제 해결

#### 헤더가 변경/삭제된 경우

```
1. 시트 열 때 경고 메시지 확인
2. 원본 헤더:
   STUDENTS: 이름, 번호, PIN해시, 토큰, 등록일, 마지막접속, 상태
   WORKS_*: 학생이름, 학생번호, 작품데이터, 생성일, 수정일, 완료여부, 상태
3. 헤더 복원 후 저장
```

#### 실수로 민감 데이터 수정한 경우

```
[PIN해시 수정한 경우]
→ 해당 학생 PIN 재설정 필요 (웹 대시보드)

[토큰 수정한 경우]
→ 해당 학생 QR 재발급 필요 (웹 대시보드)

[pinSalt 수정한 경우]
→ 복구 불가, 모든 학생 PIN 재설정 필요
→ 백업이 있다면 SETTINGS 시트 복원
```
