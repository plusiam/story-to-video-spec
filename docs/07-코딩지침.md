# 스토리 구성 웹학습지 - 코딩 지침

## 개요

이 문서는 바이브 코딩 방식으로 개발할 때 참조할 **구체적인 프롬프트 라이브러리**와 **기술 명세**를 정의한다. 협의된 9가지 결정 사항을 프롬프트로 변환하여 AI에게 일관된 코드를 생성하도록 한다.

**관련 문서:**
- `00-바이브코딩-지침.md` - 바이브 코딩 철학과 방법론
- `auth-system-v2/` - 상세 설계 문서

## 1. 프로젝트 컨텍스트 (모든 프롬프트 공통)

모든 프롬프트 앞에 붙일 공통 맥락:

```
[프로젝트 컨텍스트]
- 이름: 스토리 구성 웹학습지
- 대상: 초등학생 (4~6학년) + 교사
- 플랫폼: Google Apps Script (GAS)
- 데이터베이스: Google Spreadsheet
- 아키텍처: 점조직 모델 (교사별 독립 시스템)
- 프론트엔드: GAS 다중 파일 + include 패턴
- 인증: PIN(6자리) + QR 토큰
- 언어: 한글 (에러 메시지, UI 모두)
```

## 2. 데이터 구조 명세

### 2.1 시트 구조 (프롬프트용 레퍼런스)

```
[STUDENTS 시트]
컬럼: 이름, 번호, PIN해시, 토큰, 등록일, 마지막접속, 상태
- 이름: 문자열 (필수)
- 번호: 숫자 1~100 (필수)
- PIN해시: SHA-256 해시 또는 빈값
- 토큰: "tk_" + UUID 12자리 (QR용)
- 등록일: Date
- 마지막접속: Date 또는 빈값
- 상태: "pending" | "active" | "inactive"

[WORKS_STEP1/2/3 시트]
컬럼: 학생이름, 학생번호, 작품데이터, 생성일, 수정일, 완료여부, 상태
- 작품데이터: JSON 문자열
- 완료여부: TRUE/FALSE
- 상태: "draft" | "submitted" | "published"

[SETTINGS 시트]
형식: Key-Value (A열: 키, B열: 값)
키 목록: teacherName, schoolName, className, pinSalt, version, createdAt, welcomeMessage, theme
```

### 2.2 JSON 구조 (작품 데이터)

```json
// WORKS_STEP1 (4컷 스토리)
{
  "title": "작품 제목",
  "panels": [
    { "id": 1, "stage": "beginning", "stageName": "발단", "description": "...", "dialogue": "..." },
    { "id": 2, "stage": "development", "stageName": "전개", "description": "...", "dialogue": "..." },
    { "id": 3, "stage": "climax", "stageName": "절정", "description": "...", "dialogue": "..." },
    { "id": 4, "stage": "ending", "stageName": "결말", "description": "...", "dialogue": "..." }
  ],
  "version": 1
}

// WORKS_STEP2 (장면 확장) - 각 패널을 여러 씬으로 확장
// WORKS_STEP3 (콘티) - 스토리보드 형태로 확장
```

## 3. 프롬프트 라이브러리

### 3.1 인증 관련

#### 프롬프트: PIN 해시 함수

```
[목표]
PIN을 SHA-256으로 해시하는 GAS 함수

[요구사항]
1. SETTINGS 시트에서 pinSalt 가져오기
2. salt + pin 결합 후 SHA-256 해시
3. Base64 인코딩하여 반환
4. salt 없으면 새로 생성하여 SETTINGS에 저장

[함수 시그니처]
function hashPin(pin: string): string

[의존성]
- getSalt(): SETTINGS에서 pinSalt 조회/생성
```

#### 프롬프트: 학생 로그인 (PIN 방식)

```
[목표]
이름+번호+PIN으로 학생 로그인하는 함수

[요구사항]
1. STUDENTS 시트에서 이름+번호로 학생 찾기
2. 학생 없음 → { success: false, error: "등록되지 않은 학생입니다." }
3. 상태가 pending → { success: false, error: "PIN 설정이 필요합니다.", needSetPin: true }
4. 상태가 inactive → { success: false, error: "비활성화된 계정입니다." }
5. PIN 해시 비교 → 불일치 시 { success: false, error: "PIN이 올바르지 않습니다." }
6. 성공 시 마지막접속 업데이트 → { success: true, name, number }

[함수 시그니처]
function loginStudent(name: string, number: number, pin: string): LoginResult
```

#### 프롬프트: 학생 로그인 (QR/토큰 방식)

```
[목표]
토큰으로 학생 바로 로그인하는 함수 (QR 스캔용)

[요구사항]
1. STUDENTS 시트에서 토큰으로 학생 찾기 (컬럼 인덱스 3)
2. 토큰 없음 → { success: false, error: "유효하지 않은 QR입니다." }
3. 상태가 active 아님 → { success: false, error: "사용할 수 없는 계정입니다." }
4. 성공 시 마지막접속 업데이트 → { success: true, name, number }

[함수 시그니처]
function loginStudentByToken(token: string): LoginResult

[특징]
- PIN 검증 없이 토큰만으로 인증
- 교실에서 QR 스캔 시 바로 입장
```

#### 프롬프트: PIN 설정 (첫 접속)

```
[목표]
pending 상태 학생이 처음 PIN을 설정하는 함수

[요구사항]
1. 이름+번호로 학생 찾기
2. 학생 없음 → 에러
3. 상태가 pending 아님 → { success: false, error: "이미 PIN이 설정되어 있습니다." }
4. PIN 해시 생성 후 저장
5. 마지막접속 업데이트
6. 상태를 active로 변경

[함수 시그니처]
function setStudentPin(name: string, number: number, pin: string): Result
```

### 3.2 학생 관리 관련

#### 프롬프트: 학생 등록 (교사용)

```
[목표]
교사가 학생을 등록하는 함수 (선등록 방식)

[요구사항]
1. 이름+번호 중복 확인 → 중복 시 에러
2. 토큰 자동 생성: "tk_" + UUID 앞 12자리
3. PIN 파라미터가 있으면 해시하여 저장, 상태 active
4. PIN 없으면 PIN해시 빈값, 상태 pending
5. 등록일 = 현재 시각
6. 마지막접속 = 빈값

[함수 시그니처]
function registerStudentByTeacher(name: string, number: number, pin?: string): RegisterResult

[반환값]
{ success: true, token: "tk_xxx", status: "pending" | "active" }
```

#### 프롬프트: 학생 일괄 등록 (CSV)

```
[목표]
CSV 데이터로 학생 일괄 등록

[요구사항]
1. CSV 파싱 (첫 행 = 헤더, 건너뛰기)
2. 각 행: [이름, 번호]
3. pinMode 옵션:
   - "student": PIN 빈값, 학생이 직접 설정 (기본값)
   - "default": PIN = 번호를 6자리로 패딩 (예: 5 → "000005")
4. 중복 학생은 건너뛰고 에러 목록에 추가
5. 결과 반환: { success: 성공 수, skipped: 건너뛴 수, errors: [...] }

[함수 시그니처]
function importStudents(csvData: string, pinMode?: "student" | "default"): ImportResult
```

#### 프롬프트: PIN 재설정 (교사용)

```
[목표]
교사가 학생 PIN을 재설정하는 함수

[요구사항]
1. 이름+번호로 학생 찾기
2. 새 PIN 해시하여 저장
3. pending 상태였다면 active로 변경
4. 토큰은 유지 (QR 재발급 불필요)

[함수 시그니처]
function resetStudentPin(name: string, number: number, newPin: string): Result
```

#### 프롬프트: QR 토큰 재발급 (교사용)

```
[목표]
기존 QR이 유출되었을 때 새 토큰 발급

[요구사항]
1. 이름+번호로 학생 찾기
2. 새 토큰 생성: "tk_" + UUID 앞 12자리
3. 토큰 컬럼 업데이트
4. 새 토큰 반환

[함수 시그니처]
function regenerateToken(name: string, number: number): TokenResult

[주의]
- 기존 QR 코드는 무효화됨
- 새 QR 다운로드 안내 필요
```

### 3.3 작품 관리 관련

#### 프롬프트: 작품 저장/업데이트

```
[목표]
학생 작품 저장 (신규 생성 또는 기존 업데이트)

[요구사항]
1. step 파라미터로 시트 결정: WORKS_STEP1, WORKS_STEP2, WORKS_STEP3
2. 학생이름+학생번호로 기존 작품 찾기
3. 있으면 업데이트 (수정일 갱신)
4. 없으면 새로 추가 (생성일, 수정일 모두 현재)
5. workData는 JSON.stringify 후 저장

[함수 시그니처]
function saveWork(studentName: string, studentNumber: number, step: 1|2|3, workData: object): SaveResult

[반환값]
{ success: true, isNew: boolean, savedAt: Date }
```

#### 프롬프트: 작품 조회

```
[목표]
학생의 특정 단계 작품 조회

[요구사항]
1. step으로 시트 결정
2. 학생이름+학생번호로 작품 찾기
3. 작품데이터 JSON 파싱하여 반환
4. 없으면 null 반환

[함수 시그니처]
function getWork(studentName: string, studentNumber: number, step: 1|2|3): WorkData | null
```

#### 프롬프트: 작품 내보내기 (JSON)

```
[목표]
학생 작품을 AI 친화적 JSON으로 내보내기

[요구사항]
1. 기본 작품 데이터 포함
2. 메타데이터 추가: title, author, created
3. ai_prompt_suggestions 필드 추가:
   - image_generation: 이미지 생성용 프롬프트 제안
   - story_expansion: 스토리 확장용 프롬프트 제안

### 3.5 UI 컴포넌트 관련

#### 프롬프트: 학생 로그인 화면

```
[목표]
학생 로그인 HTML 페이지

[바이브]
- 초등학생 친화적, 밝고 따뜻한 느낌

2. 환영 메시지 (SETTINGS.welcomeMessage)
3. 입력 폼:
   - 이름 (text)
   - 번호 (number, 1-100)
   - PIN (password, 6자리 숫자)
4. 로그인 버튼
5. 에러 메시지 영역 (빨간색 박스)

[스타일]
- 메인 색상: #4A90D9
- 배경: #F5F5F5
- 카드: 흰색, 둥근 모서리, 그림자
- 글꼴: Noto Sans KR, 16px 이상

#### 프롬프트: 교사 초기 설정 마법사

```
[목표]
처음 접속한 교사를 위한 5단계 설정 마법사

[조건]
- SETTINGS.teacherName이 비어있을 때 자동 표시

[단계]
1. 환영 & 개요 (2분 예상 안내)
2. 선생님 정보 (이름 입력, 이메일 자동)
3. 학급 정보 (학교, 학급명, 환영 메시지)
4. 학생 등록 방법 안내 (3가지 방법 설명)
5. 완료 (URL 표시, QR 버튼, 다음 단계 안내)

[바이브]
- 친절하고 차근차근
- 각 단계 진행률 표시 (1/3, 2/3...)
- 이전/다음 버튼
```

#### 프롬프트: 교사 대시보드

```
[목표]
교사용 메인 대시보드

[구성]
1. 헤더: 선생님 이름, 학교/학급, 설정 버튼

   - 정렬: 번호순
   - 액션: QR 다운로드, PIN 재설정, 상세보기
4. 플로팅 버튼: 학생 추가
5. 하단: 학습지 URL 표시, 복사/QR/공유 버튼

[빈 상태]
- 학생 0명일 때: 빠른 시작 가이드 표시
- "학생 1명 등록하기", "명단 일괄 등록하기" 버튼

## 4. GAS 파일 구조

### 4.1 백엔드 (.gs 파일)

```
📁 Google Apps Script 프로젝트
├── Code.gs           // 메인 (doGet, include)
├── Auth.gs           // 인증 (로그인, PIN)
├── Student.gs        // 학생 CRUD
├── Work.gs           // 작품 CRUD
├── QR.gs             // QR 생성 (프론트 지원)

## 5. 보안 체크리스트

### 5.1 반드시 검토할 항목

| 항목 | 검토 내용 | 담당 |
|------|----------|------|
| PIN 해시 | salt 사용 여부, SHA-256 정상 동작 | 직접 확인 |
| 토큰 생성 | 충분한 랜덤성, 중복 가능성 | 직접 확인 |
| 입력값 검증 | 이름/번호/PIN 형식 검사 | AI + 검토 |
| 에러 메시지 | 민감 정보 노출 여부 | 직접 확인 |
| 권한 검사 | 학생-교사 구분 로직 | 직접 확인 |

### 5.2 수정하면 안 되는 컬럼 (교사 안내용)

```
STUDENTS 시트:
⚠️ PIN해시 - 직접 수정 시 로그인 불가
⚠️ 토큰 - 직접 수정 시 QR 무효화
⚠️ 등록일 - 자동 기록, 수정 불필요
⚠️ 마지막접속 - 자동 기록, 수정 불필요

SETTINGS 시트:
⚠️ pinSalt - 변경 시 모든 학생 로그인 불가
⚠️ version - 업데이트 관리용
⚠️ createdAt - 시스템 생성 기록
```
