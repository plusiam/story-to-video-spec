## 타임라인

### Phase 1: 초기 기획 (학급 모드 단일 시스템)

**목표**: 초등학생 4컷 스토리 구성 웹학습지

**초기 접근**:
- 중앙 집중형 GAS 시스템
- 모든 교사/학생이 하나의 시스템 공유

**문제 발견**:
- GAS 동시접속 제한 (약 100명)
- 대규모 사용 시 장애 발생

### Phase 2: 점조직 모델 도입

**해결책**: 각 교사가 독립된 시스템 운영

```
[기존: 중앙 집중형]           [변경: 점조직 분산형]
      ┌───────┐               ┌───────┐ ┌───────┐ ┌───────┐
      │ 중앙  │               │ 김쌤  │ │ 박쌤  │ │ 이쌤  │
      │ GAS  │               │ 독립  │ │ 독립  │ │ 독립  │
      └───────┘               └───────┘ └───────┘ └───────┘
         │                       │         │         │
    모든 학생               30명      25명      28명
```

**결정 사항**:
- 템플릿 "사본 만들기" 방식
- 교사별 완전 독립 운영
- 데이터는 교사 드라이브에만 저장

→ 상세: [docs/auth-system-v2/04-점조직모델.md](./docs/auth-system-v2/04-점조직모델.md)

### Phase 3: 학생 인증 시스템 설계

**결정 사항**:

| 항목 | 결정 |
|------|------|
| 가입 방식 | 교사 선등록 필수 |
| PIN | 6자리 숫자, SHA-256 해시 |
| QR 로그인 | 학생별 영구 토큰 (PIN 불필요) |
| 상태 관리 | pending → active → inactive |

**학생 로그인 흐름**:
```
[교실] QR 스캔 → 바로 입장
[집]   URL 접속 → 이름+번호+PIN → 입장
```

→ 상세: [docs/auth-system-v2/01-기획문서.md](./docs/auth-system-v2/01-기획문서.md)

### Phase 4: 기술 결정

**프론트엔드 아키텍처**:
- GAS 다중 파일 + include 패턴
- 외부 서버 의존 없음 (점조직 독립성 유지)

**교사 설정 방식**:
- 하이브리드: 핵심은 웹 UI + 고급은 시트 직접

**QR 코드**:
- 반 전체 QR + 학생별 QR
- JS 라이브러리로 생성 (외부 의존 없음)
- PDF 다운로드 (4명/페이지)

→ 상세: [docs/auth-system-v2/05-추가협의사항.md](./docs/auth-system-v2/05-추가협의사항.md)

### Phase 5: 개인 모드 분리 (2026-01-17)

**새로운 요구사항**:
- 학급이 아닌 개인 사용자 대상
- 계정 기반 인증
- AI 기능 추가 예정

**결정: 두 시스템 완전 분리**

| 학급 모드 | 개인 모드 |
|----------|----------|
| GAS + Spreadsheet | React + Supabase |
| 점조직 (분산) | 중앙 집중식 |
| PIN + QR | Google/이메일 로그인 |
| 교사 선등록 | 관리자 승인 |

**개인 모드 승인 시스템**:
```
MVP: 자동 승인 (VITE_AUTO_APPROVE_USERS=true)
Production: 관리자 수동 승인
```

**폴더 분리**:
```
story-creator/
├── classroom-gas/    # 학급 모드 (기존)
├── personal-app/     # 개인 모드 (신규)
└── landing/          # 통합 랜딩 페이지
```

**목적**: AI 지원 배포 용이성 확보

| 이슈 분산 | 메인 저장소가 이슈 허브 |

### 점조직 모델 (학급 모드)

```
📦 원본 템플릿 (classroom-gas/)
          │
          │ "사본 만들기"
          ├─────────────────────────────────────┐
          │                                     │
          ▼                                     ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ 김쌤 시스템     │  │ 박쌤 시스템     │  │ 이쌤 시스템     │
│ (완전 독립)    │  │ (완전 독립)    │  │ (완전 독립)    │
└─────────────────┘  └─────────────────┘  └─────────────────┘
```

- 각 교사가 템플릿을 복사하여 **자신만의 독립 시스템** 운영
- 데이터는 **교사 본인 드라이브**에만 저장
- 중앙 서버 의존 없음 → **동시접속 제한 문제 해결**

### 개인 모드 승인 시스템

```
MVP 단계 (현재):
  가입 → 자동 승인 (VITE_AUTO_APPROVE_USERS=true)

Production 단계 (나중):
  가입 신청 → pending → 관리자 승인 → approved

※ 승인 구조는 이미 설계 완료 (환경변수만 변경)
```

**사용자 상태 (Supabase users.status)**
| 상태 | 설명 |
|------|------|
| `pending` | 승인 대기 |
| `approved` | 승인됨 (사용 가능) |
| `rejected` | 거절됨 |
| `suspended` | 정지됨 |
