# 개발 전략: 저장소 분리

> **작성일**: 2026-01-17
> **목적**: 학급 모드 독립 배포를 위한 저장소 분리 전략

---

## 🎯 분리 목적

### 핵심 목표
1. **교사의 AI 지원 배포 용이성** - 학급 모드 코드만 별도 저장소에서 AI 도움받아 쉽게 복사/배포
2. **독립적 버전 관리** - 각 모드별 독자적 개발 사이클
3. **단순화된 기여** - 학급 모드 기여자는 GAS만 알면 됨

### 사용자 경험
- **외부에서는 하나의 서비스로 보임** (통합 랜딩 페이지)
- 내부적으로는 완전히 독립된 두 시스템

---

## 📦 저장소 구조

### 저장소 1: `story-worksheet` (메인)

```
story-worksheet/
├── landing/              # 통합 랜딩 페이지 (GitHub Pages)
│   └── index.html
├── personal-app/         # 개인 모드 (React + Supabase)
│   ├── src/
│   ├── supabase/
│   └── README.md
├── docs/                 # 기획 문서 아카이브
├── .github/              # Issue/PR 템플릿 (이슈 허브)
│   └── ISSUE_TEMPLATE/
│       ├── bug_report.md
│       ├── feature_request.md
│       └── classroom_mode.md   # → 학급 모드 저장소로 안내
├── README.md             # 프로젝트 전체 개요
├── HISTORY.md            # 개발 히스토리
└── DEVELOPMENT_STRATEGY.md  # 이 문서
```

**역할**:
- 통합 랜딩 페이지 호스팅 (GitHub Pages)
- 개인 모드 개발 및 배포 (Vercel)
- 이슈/PR 허브 (학급 모드 이슈도 여기서 트리아지)

### 저장소 2: `story-worksheet-classroom` (학급 모드 전용)

```
story-worksheet-classroom/
├── Code.gs               # GAS 메인 진입점
├── Auth.gs               # 인증 시스템
├── Student.gs            # 학생 관리
├── Work.gs               # 작품 CRUD
├── Settings.gs           # 설정
├── Utils.gs              # 유틸리티
├── *.html                # 프론트엔드 파일들
├── README.md             # 교사용 설치 가이드
└── SETUP-WITH-AI.md      # AI 도움 설치 가이드
```

**역할**:
- GAS 코드만 포함 (단순화)
- 교사가 AI에게 "이 저장소로 학급용 스토리 학습지 설치해줘" 요청 가능
- 독립적 버전/릴리즈

## 🛡️ 우려 사항 해결

### 1. 랜딩 페이지 위치
| 우려 | 해결 |
|-----|------|
| 어디에 둘지 혼란 | **메인 저장소에만 유지** |
| 중복 관리 부담 | 학급 모드 저장소는 랜딩 없음 |

### 2. 버전 동기화
| 우려 | 해결 |
|-----|------|
| 두 저장소 버전 맞추기 어려움 | **동기화 불필요** - 완전 독립 시스템 |
| 의존성 없음 | 학급 모드는 개인 모드에 의존하지 않음 |

### 3. 이슈/PR 관리 분산
| 우려 | 해결 |
|-----|------|
| 어디에 이슈 올릴지 혼란 | **메인 저장소가 이슈 허브** |
| 학급 모드 이슈 | 템플릿으로 안내 → 필요시 이동 |

### 4. 문서 중복
| 우려 | 해결 |
|-----|------|
| README 중복 작성 | **역할 특화** |
| 메인 README | 전체 프로젝트 개요 |
| 학급 README | 교사용 설치 가이드 (다른 목적) |
