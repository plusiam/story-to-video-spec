## 🪄 또리 AI 도우미 핵심 개념

### 설계 철학

```
❌ 기존 생각 (순차적, 외부 도구)
```

## 🔧 AI 연동 방식 (이원화)

### Personal 모드

| 항목 | 내용 |
|------|------|
| **AI 호출** | Supabase Edge Function → Gemini API |
| **API 키** | 운영자가 환경변수로 관리 |
| **비용** | 운영자 부담 (사용량 모니터링) |
| **장점** | 완벽한 UX, 사용자 설정 불필요 |

### Classroom 모드

| 항목 | 내용 |
|------|------|
| **AI 호출** | GAS 서버사이드 → Gemini API |
| **API 키** | 교사가 설정 (PropertiesService 암호화 저장) |
| **비용** | 교사 부담 (월 ~100원 미만) |
| **장점** | 점조직 독립성 유지, 무료 |

### API 키 보안 (Classroom)

```javascript
// ✅ 안전한 방식: GAS PropertiesService
// 교사가 설정할 때
PropertiesService.getScriptProperties().setProperty('GEMINI_API_KEY', key);

// API 호출할 때 (서버 사이드에서만)
function callGeminiAPI(prompt) {
  const key = PropertiesService.getScriptProperties().getProperty('GEMINI_API_KEY');
  // 학생 브라우저에서는 이 키를 절대 볼 수 없음
  const response = UrlFetchApp.fetch('https://generativelanguage.googleapis.com/...', {
    headers: { 'x-goog-api-key': key },
    payload: JSON.stringify({ contents: [{ parts: [{ text: prompt }] }] })
  });
  return JSON.parse(response.getContentText());
}
```

## 📊 데이터베이스 스키마

### Supabase (Personal 모드)

#### users 테이블 (기존)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK, auth.users 참조 |
| email | text | 이메일 (unique) |
| nickname | text | 닉네임 |
| status | text | pending/approved/rejected/suspended |
| role | text | user/admin |

#### works 테이블 (기존)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| user_id | uuid | FK → users |
| title | text | 작품 제목 |
| step | int | 진행 단계 (1-3) |
| panels | jsonb | 4컷 패널 데이터 |
| source | text | 'manual' / 'ttori' |
| status | text | draft/complete |
| created_at | timestamp | 생성일 |
| updated_at | timestamp | 수정일 |

#### ai_sessions 테이블 (신규) ⭐

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| work_id | uuid | FK → works |
| user_id | uuid | FK → users |
| title | text | 세션 제목 (자동 생성) |
| messages | jsonb | 대화 기록 배열 |
| created_at | timestamp | 생성일 |
| updated_at | timestamp | 수정일 |

**messages JSON 구조**:
```json
[
  {
    "role": "user",
    "content": "외로운 강아지 이야기 만들어줘",
    "timestamp": "2026-01-21T10:30:00Z"

  {
    "role": "assistant",
    "content": "좋아요! 어떤 강아지인가요? 몇 가지 질문...",
    "timestamp": "2026-01-21T10:30:05Z"
  }
]
```

4. **맥락 인식**: 현재 작품 내용을 파악하고 맞춤 도움
5. **한글 우선**: 초등학생 대상, 모든 UI/프롬프트 한글화
6. **무료 우선**: Classroom은 교사 API 키로 거의 무료
7. **보안 철저**: API 키는 서버에서만 사용, 학생에게 노출 안 함

## 🪄 또리 시스템 프롬프트 (초안)

```markdown
# 또리 - 스토리 구성 AI 도우미

## 역할
너는 "또리"야. 초등학생이 4컷 스토리(기-승-전-결)를 만들 때 도와주는 친절한 AI 도우미야.

## 성격
- 밝고 친근한 말투 (반말 OK, 이모지 적절히 사용)
- 칭찬을 많이 해줘
- 어려운 단어는 쉽게 풀어서 설명해
- 학생의 아이디어를 존중하고 발전시켜줘

## 대화 방식
1. 먼저 학생이 쓴 내용을 칭찬해줘
2. 막힌 부분이 있으면 선택지(A/B/C)를 제안해줘
3. 한 번에 너무 많은 정보를 주지 마
4. 학생이 선택하면 그 방향으로 더 구체화해줘

## 제안 형식
제안할 때는 항상 이 형식을 사용해:

[제안]
A) 첫 번째 아이디어
B) 두 번째 아이디어  
C) 세 번째 아이디어

## 금지 사항
- 폭력적이거나 무서운 내용
- 슬프거나 우울한 결말
- 어른 주제 (연애, 정치 등)
- 특정 브랜드나 캐릭터 이름

## 현재 작품 정보
{현재 작품의 제목과 패널 내용이 여기에 들어감}
```
